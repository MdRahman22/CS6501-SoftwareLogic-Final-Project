# Missile Chase in Lean 4

CS6501 Software Logic Final Project

## Overview

This project is a Lean 4 missile chase simulation with a retro browser visualization.

The main goal is to model motion over time:

- `position(t)`: where an object is at time `t`
- `velocity(t)`: how position changes over time
- `acceleration(t)`: how velocity changes over time

Lean defines the model, proves simple facts about it, and exports frame data to `frames.json`. The browser page reads that data for demo mode and also includes an interactive BigBang-style game.

## How to Run

Clone the repo:

```powershell
git clone <repo-url>
cd CS6501-SoftwareLogic-Final-Project
```

Generate the Lean frame data:

```powershell
lake env lean --run Project.lean
```

Start a local server:

```powershell
python -m http.server 8000
```

Open:

```text
http://localhost:8000/game.html
```

Use the local server instead of double-clicking `game.html`, because the browser may block loading `frames.json` directly.

On Windows, you can also try:

```text
start_game.bat
```

## Lean Model

The main world state is:

```lean
structure SimState where
  tick : Nat
  targetPos : Point2
  targetVel : Vec2
  targetAcc : Vec2
  missilePos : Point2
  missileVel : Vec2
  missileAcc : Vec2
```

The main update function is:

```lean
nextState : SimState -> SimState
```

The main time function is:

```lean
stateAt init t
```

This gives the simulation state at time `t`.

The project defines time-indexed motion functions such as:

```lean
targetPositionAt init t
targetVelocityAt init t
targetAccelerationAt init t
```

Since the simulation is tick-based, derivatives are represented as finite differences:

- velocity = change in position
- acceleration = change in velocity

## BigBang-Style Game Loop

The project includes a small BigBang-style structure:

```lean
structure BigBang (World : Type) where
  onTick : World -> World
  onKey : World -> String -> World
  stopWhen : World -> Bool
```

For this project:

- `World` is `SimState`
- `onTick` is `nextState`
- `onKey` changes target velocity
- `stopWhen` checks whether the missile caught the target

## Theorems

The project proves:

- `nextState_keeps_target_velocity`: one update step keeps target velocity unchanged
- `target_velocity_constant`: target velocity is constant at every time `t`
- `target_acceleration_zero`: target acceleration is zero
- `framesUpTo_length`: frames from 0 through `n` have length `n + 1`
- `statesUpTo_length`: exported states from 0 through `n` have length `n + 1`
- BigBang wrapper facts, including that the BigBang tick is `nextState`

## Browser Modes

### Lean Demo Mode

Reads `frames.json`, which is generated by Lean.

### Interactive Mode

The player controls the green target.

Controls:

```text
Arrow keys or WASD: change direction
Space: stop movement
Pause: pause
Restart: restart
```

The player loses if the missile catches the target or the target hits the border.

The player wins by surviving to tick 200.
